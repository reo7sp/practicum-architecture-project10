## Список мер по устранению узких мест

1. Данные
   1. Выполнить очистку и дедупликацию данных.
   2. Нормализовать форматы и справочники.
   3. Определить единый источник правды по каждому домену.
   4. Автоматизировать проверку полноты переноса файлов и записей.

2. Интеграции
   1. Упростить переходные интеграции со старыми системами.
   2. Внедрить идемпотентность операций.
   3. Проверить и стабилизировать API-контракты.
   4. Буферизовать обмен с внешними системами на период миграции.

3. Релизы и переключение
   1. Подготовить cutover-план и rollback-план.
   2. Минимизировать срок двойного учёта.
   3. Зафиксировать момент окончательного переключения.

4. Производительность
   1. Провести нагрузочное тестирование.
   2. Заложить резерв CPU, RAM и диска.
   3. Ограничить число синхронных вызовов между сервисами.
   4. Проверить самые тяжёлые операции на критическом пути.

5. Доступы
   1. Подготовить матрицу ролей.
   2. Проверить сценарии доступа до cutover.
   3. Проверить перенос учётных записей и прав.

6. Команда
   1. Обучить пользователей новой системе.
   2. Провести пилотный запуск на ограниченной группе.
